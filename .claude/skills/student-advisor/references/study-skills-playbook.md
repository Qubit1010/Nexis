# Study Skills Playbook — Notes, Reading, Focus, Planning

Practical systems for capturing and processing material. Grounded in `research-synthesis.md` Q2.
Core principle: **notes and reading are only valuable if they feed retrieval practice later.** The
method and the review matter more than pen-vs-laptop.

## Note-taking: the honest evidence

- **Medium is roughly a wash** once laptop distraction is removed (Voyer et al. meta-analysis, mean
  g ≈ -0.008). Handwriting has a modest edge in some reviews, mainly because it forces **generative
  processing** — you paraphrase instead of transcribing verbatim. If you type, type *in your own
  words*, not a transcript.
- The methods that have lasted all bake in active recall + spaced review. Pick one and be consistent:

| Method | Best for | Mechanism |
|---|---|---|
| **Cornell** (Pauk) | Lectures, structured content | Cue column + notes + summary forces recall |
| **Outline** | Reading, hierarchical material | Structure by dependency |
| **Mapping / concept maps** | Connected systems, big-picture | Visual links (dual coding) |
| **Feynman** | Checking understanding | Explain it simply, find the gaps |

**Cornell in practice:** split the page — notes on the right, **cue questions** on the left (written
after class), **summary** at the bottom. Later, cover the notes and answer the cues from memory. That
turns your notes into a retrieval-practice deck.

## Active reading: SQ3R (beats highlighting)

Survey, Question, Read, Recite, Review (Francis Robinson). Before reading: skim headings and turn them
into questions. While reading: look for the answers. After each section: **recite from memory** before
moving on. This builds testing into reading instead of passively highlighting (which is low-utility,
see `learning-science.md`).

## Focus & attention

- **Single-task.** The laptop-notes penalty is really a distraction penalty — kill notifications, keep
  only the current subject's materials in view.
- **Focused blocks** of ~40-90 minutes with short breaks beat marathon sessions (also better for
  spacing).
- Design the environment so starting is frictionless (see `retention-motivation-playbook.md` for the
  2-minute rule and habit design).

## Building a study plan (template)

```
# Study Plan: [subject / goal]
Weekly anchor: [fixed days/times you'll study — tie to an existing routine]
Per session:
  - 5 min: recall last session (blank-page / self-quiz)
  - 30-60 min: new material, captured generatively (Cornell/outline)
  - 5-10 min: turn it into questions for future retrieval
Spaced review ladder: revisit each topic at day 1, 3, 7, 14 (or use Anki)
Weekly: one mixed self-test across topics (interleaving)
```

Lead with the highest-leverage habit (spaced self-testing), then layer the rest. For a time-poor
student, one disciplined self-test cycle beats an elaborate system they won't sustain. Offer the
Google Docs export for a full plan (see SKILL.md).
