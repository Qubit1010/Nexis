# Research Methodology — the light/medium/deep method

Evidence + citations in `research-synthesis.md` Q1. This governs HOW to run a research pass at each
depth, not which engine (that's `service-selection.md`).

## The loop (every depth runs a subset)
1. **Frame** a precise, answerable question + sub-questions [1][6].
2. **Search broadly** — cast a wide net before narrowing [1][2].
3. **Evaluate** each source (see credibility, below) [7][9].
4. **Triangulate** — confirm claims across independent sources [1][10].
5. **Synthesize** with citations as you go [1][6].

## Depth = how much of the loop you run
- **light** — steps 1-2-5 fast: one engine's answer + a few sources. For simple facts, quick lookups.
- **medium** — full loop, 2-3 engines fused, ranked by cross-source agreement, short synthesis. The
  default for real questions.
- **deep** — full loop + content extraction of the top pages + a structured, cited report saved to
  `research/`. For market/competitor/strategic/scientific work. Use systematic-review discipline
  (documented, reproducible search + screening) when the answer must be defensible [13][15][16].

## Evaluating a source (do this, not just CRAAP)
- **Lateral reading** beats vertical reading: to judge a source, open new tabs and see what *other*
  independent sources say about it — the move fact-checkers use and amateurs skip [8][12].
- Ask: who is behind it, what's the evidence, what do other sources say — before trusting a claim
  [9][11].
- CRAAP (Currency/Relevance/Authority/Accuracy/Purpose) is a starting checklist, not sufficient alone
  [7].

## Synthesis rules (baked into the skill)
- Cite only sources actually gathered; never invent a citation or URL [98].
- Lead with the finding, then the evidence. Flag disagreements and gaps honestly ("sources don't
  cover X") rather than papering over them.
- When accuracy is critical, verify: re-check each claim against its cited source; drop or refine
  anything unsupported (Chain-of-Verification / post-hoc citation) [100][101].
