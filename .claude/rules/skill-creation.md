# Skill Creation

## When to use the skill-creator

When Aleem asks to **create, build, or make a new skill** (or rebuild/redo one from scratch):

- **He explicitly names the skill-creator** ("use skill-creator", "with skill-creator", "via the skill creator", "/skill-creator") → proceed with the `skill-creator` workflow. No question needed.
- **He does NOT mention the skill-creator** → **ask first** before creating anything:
  > "Want me to build this with the `skill-creator` workflow (proper SKILL.md + evals + iteration), or just scaffold it directly?"
  Wait for his answer, then proceed accordingly.

Default recommendation when asking: the `skill-creator` workflow, since it produces a properly structured, testable skill consistent with the rest of `.claude/skills/`.

This is a conversational judgment rule for Claude, not an automated hook. Apply it to new-skill requests only — editing or improving an existing skill doesn't require the question (though the skill-creator handles that too).
