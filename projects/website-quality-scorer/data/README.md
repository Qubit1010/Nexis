# Dataset

## Cold Start (v1)
v1 model is trained on a synthetic dataset generated by `backend/ml/synthetic_dataset.py`. Labels come from the heuristic scorer (`backend/ml/heuristic_label.py`). No real data needed to deploy.

## Real Labeled Dataset (v2)
Target: 300-500 manually labeled websites, dual-annotator scoring.

### Labeling Rubric

Each website is scored across 4 dimensions, 0-25 per dimension. Composite score = sum (0-100).

| Score | UX | Content | Technical | Trust |
|---|---|---|---|---|
| **0-10** Poor | No clear CTA, confusing nav, unresponsive layout | Unreadable, generic, no value prop | PageSpeed <50, no HTTPS, broken | No social proof, no contact info |
| **11-17** Average | Basic layout, some CTAs, partial mobile support | Readable but generic, weak hierarchy | PageSpeed 50-74, HTTPS, slow | Some signals (1-2), thin contact |
| **18-25** Good | Clear hierarchy, strong above-fold CTA, fully responsive | Clear copy, value prop visible, good headings | PageSpeed 75+, fast, secure, schema | Reviews + logos + contact + privacy + pricing |

### Inter-Rater Reliability
- Both annotators score independently
- Cohen's Kappa target ≥ 0.70
- Disagreements >5 points per dimension resolved via discussion
- Final label = mean of both annotators (after resolution)

### File Format
`labeled/dataset.csv` columns:
- `url` — website URL
- `feature_*` — 41 columns matching `backend/features/extractor.FEATURE_NAMES`
- `score_ux`, `score_content`, `score_technical`, `score_trust` — per-dimension labels (0-25)
- `score_total` — composite (0-100)
- `annotator_1`, `annotator_2` — initials of both annotators
- `kappa_agreement` — boolean (resolved or original agreement)

### Industry Distribution
Target sample distribution (~25% each):
- E-commerce
- SaaS / B2B software
- Professional services / agencies
- Portfolios / personal brands
